# TFTBlog MVP 后续 4 步计划

来源：用户验收本地 MVP 后，要求将下一步计划完整落到 CSV。

批准状态：用户已明确使用 `mission-approved-doc` 要求转换为 CSV。

## Goal

把当前已通过本地验收的移动端攻略杂志 MVP 推进到可上线、可持续导入内容的版本。

## Scope

1. 补齐图片资产链路。
2. 把本地 MVP 部署到服务器。
3. 打磨移动端详情页。
4. 做内容导入规范。

## Constraints

- 不做后台文章编辑器。
- 不做审核流。
- 不做高操作量重功能。
- 服务器内存较小，避免在服务器直接执行 Next build。
- 内容来源仍以 Obsidian Markdown 为主。

## Plan

### 1. 补齐图片资产链路

定位 Obsidian 附件目录，做导入脚本：复制 Markdown 时自动复制 `![[图片.png]]` 引用图，并转换成站内可访问路径。完成后攻略详情页不再出现“图片待整理”占位。

### 2. 把本地 MVP 部署到服务器

当前线上仍是维护页。选择适合低内存服务器的部署方式，本地构建后通过 SCP 上传产物并切换 PM2/Nginx 服务，避免服务器侧 build。

### 3. 打磨移动端详情页

围绕手机真实阅读体验检查标题、标签、图片宽度、段落节奏、目录横滑、返回路径和长文滚动表现。

### 4. 做内容导入规范

约定每篇攻略最少字段：标题、标签、封面、来源、更新时间。后续只需要在 Obsidian 写文章，通过脚本同步到站点，不依赖后台编辑器。

## Validation

- `npm run lint` 通过。
- `npm run build` 通过。
- 本地浏览器可访问首页、攻略列表、攻略详情、资料入口。
- 线上域名可访问新 MVP。
- 至少 3 篇攻略详情页图片正常显示。
